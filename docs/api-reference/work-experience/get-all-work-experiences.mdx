## Endpoint

```
GET /api/v1/job-seekers/me/work-experiences
```

**Base URL**: `http://localhost:3000/api/v1`

<Note>This endpoint requires **authentication** and **Job Seeker role**.</Note>

## Authentication

Include the access token in the Authorization header:

```bash
Authorization: Bearer YOUR_ACCESS_TOKEN
```

## Description

Retrieve all work experience entries for the authenticated job seeker. The results are ordered by start date (most recent first).

### Response Fields

<ResponseField name="id" type="string">
  Unique identifier for the work experience entry.
</ResponseField>

<ResponseField name="companyName" type="string">
  Name of the company.
</ResponseField>

<ResponseField name="jobTitle" type="string">
  Title of the position.
</ResponseField>

<ResponseField name="description" type="string | null">
  Job description and responsibilities.
</ResponseField>

<ResponseField name="location" type="string | null">
  Work location.
</ResponseField>

<ResponseField name="startDate" type="string">
  Start date (ISO 8601 format).
</ResponseField>

<ResponseField name="endDate" type="string | null">
  End date (ISO 8601 format). Null if currently working.
</ResponseField>

<ResponseField name="isCurrent" type="boolean">
  Whether this is the current job.
</ResponseField>

## Error Responses

### 401 Unauthorized - Not Authenticated

```json
{
  "success": false,
  "error": {
    "message": "Unauthorized",
    "statusCode": 401,
    "timestamp": "2026-02-18T10:00:00.000Z",
    "path": "/job-seekers/me/work-experiences",
    "method": "GET"
  }
}
```


## Notes

<Info>**Empty Array**: If no work experiences exist, an empty array is returned.</Info>

<Tip>
  **Sorting**: Results are automatically sorted by start date in descending order (most recent
  first).
</Tip>
