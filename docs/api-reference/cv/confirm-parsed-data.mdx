## Endpoint

```
POST /api/v1/cv-parse/confirm
```

**Base URL**: `http://localhost:3000/api/v1`

<Note>This endpoint requires **authentication** and **Job Seeker role**.</Note>

## Authentication

Include the access token in the Authorization header:

```bash
Authorization: Bearer YOUR_ACCESS_TOKEN
```

## Description

Review the parsed CV data and confirm to save it to your profile. This creates or updates your JobSeekerProfile, Education, WorkExperience, and JobSeekerSkill records. You can modify any fields in the request body before confirming.

## Request Body

<ParamField body="personalInfo" type="object" required>
  Personal information including name and contact details.
</ParamField>

<ParamField body="personalInfo.firstName" type="string" required>
  First name of the job seeker.
</ParamField>

<ParamField body="personalInfo.lastName" type="string" required>
  Last name of the job seeker.
</ParamField>

<ParamField body="personalInfo.email" type="string" required>
  Email address.
</ParamField>

<ParamField body="personalInfo.phone" type="string" required>
  Phone number.
</ParamField>

<ParamField body="personalInfo.location" type="string" optional>
  Location/address.
</ParamField>

<ParamField body="personalInfo.linkedinUrl" type="string" optional>
  LinkedIn profile URL.
</ParamField>

<ParamField body="personalInfo.githubUrl" type="string" optional>
  GitHub profile URL.
</ParamField>

<ParamField body="personalInfo.portfolioUrl" type="string" optional>
  Portfolio website URL.
</ParamField>

<ParamField body="title" type="string" required>
  Job title (e.g., "Software Engineer").
</ParamField>

<ParamField body="summary" type="string" required>
  Professional summary.
</ParamField>

<ParamField body="education" type="array" required>
  Array of education entries.
</ParamField>

<ParamField body="workExperience" type="array" required>
  Array of work experience entries.
</ParamField>

<ParamField body="skills" type="array" required>
  Array of skills. Each skill has `name` and `verified` flag.
</ParamField>

<ParamField body="profile" type="object" required>
  Profile preferences and details.
</ParamField>

<ParamField body="profile.expectedSalary" type="number" optional>
  Expected salary.
</ParamField>

<ParamField body="profile.workPreference" type="string" optional>
  Work preference: ONSITE, REMOTE, HYBRID, ANY
</ParamField>

<ParamField body="profile.yearsOfExperience" type="number" optional>
  Years of experience.
</ParamField>

<ParamField body="profile.noticePeriod" type="number" optional>
  Notice period in days.
</ParamField>

<ParamField body="profile.availabilityStatus" type="string" optional>
  Availability status: OPEN_TO_WORK, NOT_LOOKING, PASSIVELY_LOOKING
</ParamField>

## Complete CV Flow

<Steps>
  <Step title="Upload CV">Call POST /cv/presigned-url to get an upload URL</Step>
  <Step title="Upload File">Upload the file using PUT with the upload URL</Step>
  <Step title="Confirm Upload">Call POST /cv/confirm to trigger NLP parsing</Step>
  <Step title="Preview Data">Call GET /cv-parse/preview to see parsed results</Step>
  <Step title="Confirm & Save">Call POST /cv-parse/confirm to save to your profile</Step>
</Steps>

<Info>
  **What gets saved**: This endpoint saves the following to your profile:
  - JobSeekerProfile: title, summary, phone, email, location, social links, work preferences
  - Education: all education entries
  - WorkExperience: all work experience entries
  - JobSeekerSkill: all skills with verified: true flag
</Info>
