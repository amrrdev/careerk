# Welcome to CareerK API

CareerK is a comprehensive job platform API that connects job seekers with employers. This documentation provides everything you need to integrate CareerK into your frontend application.

## Base URL

```
http://localhost:3000/api/v1
```

All authentication endpoints are under the `/api/v1` prefix.

For production, replace with your actual API URL.

## Authentication

CareerK uses JWT (JSON Web Tokens) for authentication. After registration or login, you'll receive:

- **Access Token**: Used to authenticate API requests (valid for 1 hour)
- **Refresh Token**: Stored in HTTP-only cookie, used to get new access tokens (valid for 24 hours)

### Using Access Tokens

Include the access token in the Authorization header:

```bash
Authorization: Bearer YOUR_ACCESS_TOKEN
```

## Quick Start

### 1. Register a User

**Endpoint**: `POST /api/v1/auth/register/job-seeker`

**Request Shape**:

```typescript
interface RegisterJobSeekerDto {
  email: string;
  password: string;
  firstName: string;
  lastName: string;
}
```

**Response Shape**:

```typescript
interface RegisterJobSeekerResponse {
  success: boolean;
  data: {
    email: string;
  };
  message: string;
}
```

### 2. Verify Email

Check your email for the 6-digit OTP code.

**Endpoint**: `POST /api/v1/auth/verify-email`

**Request Shape**:

```typescript
interface VerifyEmailDto {
  email: string;
  code: string;
}
```

**Response Shape**:

```typescript
interface VerifyEmailResponse {
  success: boolean;
  data: {
    id: string;
    email: string;
    firstName: string;
    lastName: string;
    accessToken: string;
    // ... other fields
  };
  message: string;
}
```

### 3. Make Authenticated Requests

Include the access token in the Authorization header:

```typescript
Authorization: Bearer<accessToken>;
```

## Response Format

All API responses follow this structure:

### Success Response

```json
{
  "success": true,
  "data": {
    // Response data here
  },
  "message": "Operation successful"
}
```

### Error Response

```json
{
  "statusCode": 400,
  "message": "Error message or array of validation errors",
  "error": "Bad Request"
}
```

## Available Endpoints

### Authentication

- `POST /auth/register/job-seeker` - Register as job seeker
- `POST /auth/register/company` - Register as company
- `POST /auth/verify-email` - Verify email with OTP code
- `POST /auth/login` - Login to existing account
- `POST /auth/refresh-token` - Refresh access token
- `POST /auth/forgot-password` - Request password reset code
- `POST /auth/reset-password` - Reset password with OTP code

## Common HTTP Status Codes

| Code | Description                                            |
| ---- | ------------------------------------------------------ |
| 200  | Success - Request completed successfully               |
| 201  | Created - Resource created successfully                |
| 400  | Bad Request - Invalid input or validation error        |
| 401  | Unauthorized - Missing or invalid authentication       |
| 403  | Forbidden - Authenticated but not authorized           |
| 404  | Not Found - Resource doesn't exist                     |
| 409  | Conflict - Resource already exists (e.g., email taken) |
| 500  | Internal Server Error - Something went wrong           |

## User Types

CareerK supports two types of users:

### Job Seeker

Individual users looking for job opportunities

- Register at `/auth/register/job-seeker`
- Access job seeker endpoints

### Company

Employers posting job opportunities

- Register at `/auth/register/company`
- Access company endpoints

## Password Reset Flow

If you forget your password, you can reset it using a two-step process:

### 1. Request Password Reset

**Endpoint**: `POST /api/v1/auth/forgot-password`

**Request Shape**:

```typescript
interface ForgotPasswordDto {
  email: string;
}
```

**Response Shape**:

```typescript
interface ForgotPasswordResponse {
  success: boolean;
  data: {
    email: string;
  };
  message: string;
}
```

### 2. Reset Password with OTP

**Endpoint**: `POST /api/v1/auth/reset-password`

**Request Shape**:

```typescript
interface ResetPasswordDto {
  email: string;
  code: string; // OTP from email
  newPassword: string;
}
```

**Response Shape**:

```typescript
interface ResetPasswordResponse {
  success: boolean;
  data: {
    message: string;
  };
  message: string;
}
```

### 2. Reset Password with OTP

```javascript
const response = await fetch('http://localhost:3000/api/v1/auth/reset-password', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    email: 'john@example.com',
    code: '123456', // OTP from email
    newPassword: 'newSecurePassword123',
  }),
});

const data = await response.json();
// Password reset successful
// All active sessions are logged out - must login again
```

## Important Security Notes

<Warning>
  **Refresh Token Cookies**: Always use `credentials: 'include'` in your fetch requests to enable
  cookie handling. The refresh token is stored as an HTTP-only cookie and cannot be accessed via
  JavaScript.
</Warning>

<Info>
  **Password Security**: All passwords are hashed using bcrypt before storage. Passwords are never
  stored in plain text.
</Info>

<Tip>
  **OTP Codes**: All OTP codes (email verification and password reset) are valid for 10 minutes and
  can be attempted a maximum of 3 times.
</Tip>

## Token Lifetimes

| Token Type    | Lifetime   | Storage                      |
| ------------- | ---------- | ---------------------------- |
| Access Token  | 1 hour     | Memory/State (client-side)   |
| Refresh Token | 24 hours   | HTTP-only cookie (automatic) |
| OTP Code      | 10 minutes | Redis (server-side)          |
