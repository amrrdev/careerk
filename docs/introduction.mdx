# Welcome to CareerK API

CareerK is a comprehensive job platform API that connects job seekers with employers. This documentation provides everything you need to integrate CareerK into your frontend application.

## Base URL

```
http://localhost:3000/auth
```

**Note**: There is no `/api/v1` prefix. All authentication endpoints are directly under the base URL.

For production, replace with your actual API URL.

## Authentication

CareerK uses JWT (JSON Web Tokens) for authentication. After registration or login, you'll receive:

- **Access Token**: Used to authenticate API requests (valid for 1 hour)
- **Refresh Token**: Stored in HTTP-only cookie, used to get new access tokens (valid for 24 hours)

### Using Access Tokens

Include the access token in the Authorization header:

```bash
Authorization: Bearer YOUR_ACCESS_TOKEN
```

## Quick Start

### 1. Register a User

```javascript
const response = await fetch('http://localhost:3000/auth/register/job-seeker', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    email: 'john@example.com',
    password: 'password123',
    firstName: 'John',
    lastName: 'Doe'
  })
});

const data = await response.json();
// Returns: { 
//   success: true, 
//   data: { email: 'john@example.com' }, 
//   message: 'Registration successful. Please check your email to verify your account.' 
// }
// Check your email for a 6-digit OTP code
```

### 2. Verify Email

Check your email for the 6-digit OTP code, then:

```javascript
const response = await fetch('http://localhost:3000/auth/verify-email', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  credentials: 'include', // Important for cookies
  body: JSON.stringify({
    email: 'john@example.com',
    code: '123456'
  })
});

const data = await response.json();
// Returns user data with accessToken
const { accessToken } = data.data;
```

### 3. Make Authenticated Requests

```javascript
const response = await fetch('http://localhost:3000/job-seekers/profile', {
  headers: {
    'Authorization': `Bearer ${accessToken}`,
    'Content-Type': 'application/json'
  },
  credentials: 'include' // Important for refresh token cookies
});

const data = await response.json();
```

## Response Format

All API responses follow this structure:

### Success Response

```json
{
  "success": true,
  "data": { 
    // Response data here
  },
  "message": "Operation successful"
}
```

### Error Response

```json
{
  "statusCode": 400,
  "message": "Error message or array of validation errors",
  "error": "Bad Request"
}
```

## Available Endpoints

### Authentication
- `POST /auth/register/job-seeker` - Register as job seeker
- `POST /auth/register/company` - Register as company
- `POST /auth/verify-email` - Verify email with OTP code
- `POST /auth/login` - Login to existing account
- `POST /auth/refresh-token` - Refresh access token
- `POST /auth/forgot-password` - Request password reset code
- `POST /auth/reset-password` - Reset password with OTP code

## Common HTTP Status Codes

| Code | Description |
|------|-------------|
| 200 | Success - Request completed successfully |
| 201 | Created - Resource created successfully |
| 400 | Bad Request - Invalid input or validation error |
| 401 | Unauthorized - Missing or invalid authentication |
| 403 | Forbidden - Authenticated but not authorized |
| 404 | Not Found - Resource doesn't exist |
| 409 | Conflict - Resource already exists (e.g., email taken) |
| 500 | Internal Server Error - Something went wrong |

## User Types

CareerK supports two types of users:

### Job Seeker
Individual users looking for job opportunities
- Register at `/auth/register/job-seeker`
- Access job seeker endpoints

### Company
Employers posting job opportunities
- Register at `/auth/register/company`
- Access company endpoints

## Password Reset Flow

If you forget your password, you can reset it using a two-step process:

### 1. Request Password Reset

```javascript
const response = await fetch('http://localhost:3000/auth/forgot-password', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    email: 'john@example.com'
  })
});

const data = await response.json();
// Returns success message regardless of email existence (security feature)
// If email exists, user receives 6-digit OTP code via email
```

### 2. Reset Password with OTP

```javascript
const response = await fetch('http://localhost:3000/auth/reset-password', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    email: 'john@example.com',
    code: '123456', // OTP from email
    newPassword: 'newSecurePassword123'
  })
});

const data = await response.json();
// Password reset successful
// All active sessions are logged out - must login again
```

## Important Security Notes

<Warning>
  **Refresh Token Cookies**: Always use `credentials: 'include'` in your fetch requests to enable cookie handling. The refresh token is stored as an HTTP-only cookie and cannot be accessed via JavaScript.
</Warning>

<Info>
  **Password Security**: All passwords are hashed using bcrypt before storage. Passwords are never stored in plain text.
</Info>

<Tip>
  **OTP Codes**: All OTP codes (email verification and password reset) are valid for 10 minutes and can be attempted a maximum of 3 times.
</Tip>

## Token Lifetimes

| Token Type | Lifetime | Storage |
|------------|----------|---------|
| Access Token | 1 hour | Memory/State (client-side) |
| Refresh Token | 24 hours | HTTP-only cookie (automatic) |
| OTP Code | 10 minutes | Redis (server-side) |
